# Hermes Goal: Personal Productivity Agentic System

Use this wording with Hermes `/goal` when working on this project:

```text
Build a safe-first agentic personal productivity system for my Gmail, Google Calendar, and later Yahoo Mail. Hermes should act as the development partner creating the system, not as a one-off inbox triage assistant. Use live data only as read-only test/sample data unless I explicitly approve mutations. Define controlled taxonomies, sender/service registry schema, feedback protocol, alert rules, and evaluation cases in C:\hermes\PersonalProductivity. The system should separate legitimacy from personal applicability, ask me structured low-friction classification questions, learn from my answers, and eventually run scheduled briefings/alerts delivered preferably via WhatsApp.
```

## Operating interpretation

Hermes is the builder/operator of the system. The future productivity agent is the component that will manage email/calendar.

Live Gmail/Calendar access is for read-only sampling and validation until the user explicitly approves mutations. Do not treat manual inbox cleanup as the main task.

Current phase: design and validate the controlled taxonomy, sender/service registry, feedback protocol, alert rules, evaluation cases, and scheduled-agent workflow.
